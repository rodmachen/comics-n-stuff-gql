# Claude Session — 2026-03-01

## Project: comics-n-stuff-gql

### Topic: Incorporate feedback into app-strategy.md

**User request:** Apply detailed architectural and workflow feedback to `docs/plans/app-strategy.md` for the DC Decade multi-platform monorepo plan.

**Feedback addressed and changes made:**

1. **Pre-compiled YAML content** — Added `compile-homepage.ts` script to Phase 0 that generates `homepage.json`, `homepage.swift`, and `homepage.kt` from `homepage.yaml`. Removed `Yams` and `kaml` runtime YAML parsing dependencies from the tech stack since homepage content is now pre-compiled.

2. **npm workspaces declared** — Root `package.json` now explicitly uses npm workspaces for `web/` and `react-native/` to prevent duplicated or conflicting shared dependencies (e.g., `@graphql-codegen/cli`).

3. **Centralized Cloudinary URL logic** — Cloudinary URL transform strings (thumbnail, detail) are now defined in `tokens.yaml` and generated into all platform token files as constants, preventing four agents from implementing slightly different URL manipulation.

4. **E2E testing scoped out of agent work** — Testing strategy now clearly separates "Agent-built" tests (unit, component, integration) from "Manual/CI — post-agent" tests (Playwright, XCUITest, Detox/Maestro, Espresso) since Claude Code cannot interact with simulators/emulators from the CLI.

5. **Pre-configured codegen configs** — Phase 0 now creates ready-to-use codegen configs in `shared/codegen/` (codegen-web.ts, codegen-rn.ts, apollo-codegen-config.json) so agents only run generation commands rather than configuring plugins from scratch.

6. **Placeholder asset explicitly listed** — `dc-placeholder.png` added to Branding Assets section and repo structure tree, ensuring agents know to handle null `coverImageUrl` with this fallback.

7. **Agent directory lockdown** — Added explicit constraint in Phase 1: agents must not modify files outside their designated directory and must treat `shared/` as immutable, reporting errors rather than attempting fixes to root-level files.
